Troubleshooting

Compatibility notes, limitations, and practical workarounds.

Use this page for overlay, notification, and compatibility issues.

Game compatibility

  • Works best with games running in Borderless Windowed mode.
  • DirectX 9, 10, and 11 games usually need borderless mode for notifications to appear above the game window.
  • Fullscreen support is limited but often better in DirectX 12 titles.
  • The app can import existing achievements automatically when local progress files are supported.

Known limitations

  • Match privilege level between app and game (admin/admin or non-admin/non-admin) for more reliable overlay behavior.
  • If overlay shortcuts fail in a game, switch to a 3-key combo and test the Ctrl+PageUp / Ctrl+PageDown fallback.
  • Overlay drag can fail in elevated or protected game contexts; in those cases use snap positions or fine nudge shortcuts instead.
  • Native PlayStation controller support for the overlay depends on Microsoft GameInput. Without it, only XInput-compatible controllers are available.
  • Overlay z-order can vary depending on Windows, GPU driver, and composition mode.

Common workarounds

  • Toggle Disable Hardware Acceleration and keep the mode that is most stable for the user's setup.
  • If the overlay shortcut works but nothing is visible, try turning hardware acceleration back on and restart the app.
  • If Special K or another injector is active, enable Force globalShortcut for overlay. This switches overlay shortcuts to Electron globalShortcut, disables hook-based drag, and can resolve compatibility/composition issues on affected systems. The user additionally in MANY cases MUST also indicate/select this on Special K:
  • Reduce long notification durations if users report delayed unlock display.
  • Use lighter presets if backdrop blur or layered animation causes stutter.
  • Avoid running multiple overlay or injector tools while troubleshooting.
